# 분석 품질 원칙

모든 단계에 공통으로 적용합니다.

- KPI 정의가 먼저입니다
- 절대값 / 상대값 / 누적값을 섞지 않습니다
- 차트는 질문에 답해야 합니다
- 큰 입력 묶음일수록 더 많이 요약하고 더 적게 보여줍니다
- 범주가 많은 차원은 `상위 N개 + 기타` 로 줄입니다
- 열이 많은 소스는 한 번에 다 보여주지 않고 렌즈를 먼저 고릅니다
- 표본이 작으면 반드시 경고합니다
- 시간 축이 약하면 예측을 억지로 만들지 않습니다
- 임원용 메시지와 실무용 근거를 구분합니다
- 컬럼명이 같다고 같은 의미라고 단정하지 않습니다
- 행 수준, 단위, 기간 축이 맞지 않으면 교차 파일 KPI를 만들지 않습니다
- 결합이 애매하면 `join` 보다 `portfolio` 를 택합니다
- 모든 인사이트와 KPI에는 **근거가 된 소스** 를 남깁니다

## 근거 소스 표기 규칙

- 단일 파일이면 `파일명.csv` 형식으로 적습니다
- 여러 파일이면 `파일A.csv + 파일B.csv (관계)` 형식으로 적습니다
- 관계는 `compare`, `stack`, `join`, `portfolio` 중 하나로 적습니다
- 같은 파일 집합을 반복해도 축약 별칭을 새로 만들지 않습니다
- 근거 소스가 없는 인사이트, KPI, 패널은 미완성으로 봅니다
