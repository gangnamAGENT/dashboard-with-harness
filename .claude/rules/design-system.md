# 대시보드 디자인 규칙

대시보드 디자인의 기준 문서는 프로젝트 루트의 `DESIGN.md` 입니다.
이 파일은 에이전트가 바로 지킬 수 있도록 그 내용을 짧게 정리한 것입니다.

## 필수 입력

- `DESIGN.md`
- `.claude/rules/input-contract.md`
- `runs/latest/outputs/00_input_manifest.json`

## 기본 원칙

- 어두운 관제 화면 느낌을 따른다
- 작은 카드 여러 개보다 큰 표면 몇 개로 위계를 만든다
- 밝은 색은 강조와 상태 표현에만 쓴다
- 메인 차트는 1개, 보조 차트는 2개 이하로 둔다
- KPI 카드는 3~5개 사이에서만 쓴다
- 탭은 1~3개로 제한한다
- 다중 소스일 때는 출처를 숨기지 않는다

## 문구 규칙

- 페이지 키커는 1단어
- 탭 라벨은 2~4자
- 패널 제목은 2~8자
- 섹션 라벨은 한 단어 우선
- 버튼과 액션 제목은 2~6자
- 설명은 제목에 붙이지 말고 바로 아래 본문으로 내린다

## 규모별 대응 규칙

- `row_scale = tiny` → 패널 1개, 차트 1개, 표 중심
- `row_scale = small` → 패널 2개 이내, 메인 차트 1개 + 보조 차트 2개 이하
- `row_scale = medium/large/huge` → 집계 우선, 원시 행 그대로 나열 금지
- `column_scale = wide/very_wide` → 모든 열을 한 번에 보여주지 말고 렌즈를 먼저 고른다
- 범주가 많은 차원은 `상위 N개 + 기타`
- `source_count > 1` → 소스 라벨 표시, 관계가 애매하면 병렬 패널 우선

## 차트 규칙

- 순수 SVG만 사용한다
- 파이 / 도넛 / 3D 차트는 쓰지 않는다
- 범주 비교는 정렬된 바 차트를 우선한다
- 시계열은 시간 컬럼이 믿을 만할 때만 쓴다
- 예측은 시간 축과 관측치가 충분할 때만 쓴다

## 표 규칙

- 표는 가로 스크롤이 가능해야 한다
- `table_row_cap`, `table_column_cap` 을 넘기지 않는다
- 넘치는 행과 열은 샘플, 집계, 선택 열 방식으로 줄인다

## 구조 규칙

- 상단 헤드
- 요약
- 지표
- 메인
- 보조 영역
- 결정표
- 액션

## 소스 규칙

- 패널마다 근거 소스를 짧게 적는다
- `join`, `stack`, `compare`, `portfolio` 중 어떤 해석인지 남긴다
- 결합 전제가 약하면 하나의 차트에 섞지 않는다
- 여러 파일이 있어도 탭 수는 늘리지 않고 소스 수를 줄여 보여준다

## 금지

- 여러 디자인 문서를 섞어서 따르기
- 색만으로 의미 전달하기
- 탭 하나에 차트 4개 넘게 넣기
- 분석에 없는 지표를 임의로 만들기
- 설명형 제목과 긴 버튼 문구 남발하기
- 근거 소스 없이 파일을 한 그래프에 합치기
