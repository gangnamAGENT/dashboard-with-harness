# @scenario-pack

## 역할

이 단계는 항상 실행하지 않습니다.
데이터가 충분히 말해 줄 때만, **앞으로의 흐름** 또는 **변수 조정 시 어떤 차이가 생기는지** 를 정리합니다.
여러 CSV가 있을 때는 예측 대상이 되는 소스와 정렬 규칙이 먼저 분명해야 합니다.

## 언제 실행하나

- 시간 컬럼이 있고 흐름 해석이 가능할 때
- 분석 질문이 미래 예측이나 개입 효과와 직접 연결될 때
- 표본이 너무 작지 않아 방향성 판단이 가능할 때
- 여러 파일이라면 시간 축과 단위가 맞는 소스만 대상으로 삼을 수 있을 때

## 필수 입력

- `runs/latest/outputs/00_input_manifest.json`
- `runs/latest/outputs/05_analysis_report.md`
- `runs/latest/outputs/04_kpi_summary.md`
- 옵션 A가 실행되었다면 `runs/latest/outputs/01b_dataset_prep.md`
- 원본 CSV들

## 해야 할 일

1. 예측이 적절한지 먼저 판단합니다
2. 적절하지 않으면 억지 예측 대신 **가정 비교 시나리오** 로 바꿉니다
3. 적절하다면 2~3개 시나리오를 만듭니다
4. 각 시나리오의 전제, 기대 결과, 한계를 함께 적습니다
5. 어떤 소스를 대상으로 했는지와 어떤 관계를 전제로 했는지 분명히 적습니다

## 출력

`runs/latest/outputs/05b_scenario_pack.md`

## 출력 형식

```md
# 시나리오·예측 패키지

## 실행 이유
- ...

## 대상 소스와 전제
- 대상 소스:
- 관계 전제:
- 시간 축 전제:

## 적용 방식
- 예측 / 가정 비교 / 민감도 분석 중 무엇을 썼는지

## 시나리오 요약
| 시나리오 | 전제 | 기대 변화 | 해석 주의 |
|---------|------|----------|-----------|

## 대시보드에 넣을 요소
- 메인 문장:
- 시뮬레이션 변수:
- 결과 지표:

## 한계
- ...
```

## 원칙

- 예측이 약하면 약하다고 분명히 씁니다
- 억지 숫자보다 해석 가능한 시나리오가 낫습니다
- 대시보드용 보조 재료를 만드는 단계라는 점을 잊지 않습니다
- 근거 소스가 불명확한 시나리오는 만들지 않습니다
