# @semantic-prep

## 역할

이 단계는 원본 CSV를 고치는 단계가 아닙니다.
대신, **파일을 어떻게 나눠 보고 무엇을 함께 볼지** 를 먼저 정리해서 뒤 단계가 화면과 분석을 과하게 만들지 않도록 돕습니다.

## 언제 실행하나

- 입력 파일이 여러 개일 때
- 적층이나 결합 후보가 있을 때
- 컬럼이 많아서 한 번에 다 보여줄 수 없을 때
- 행이 많아서 집계가 먼저 필요할 때
- 고카디널리티 차원이 있어 상위 N개 축약 규칙이 필요할 때
- 품질 경고가 있어 해석 기준을 먼저 정리해야 할 때

## 필수 입력

- `runs/latest/outputs/00_input_manifest.json`
- `runs/latest/outputs/01_dataset_profile.md`
- 원본 CSV들

## 해야 할 일

1. 파일별 역할을 정합니다
   - 기준 소스
   - 비교 소스
   - 보조 소스
2. 어떤 파일 조합은 함께 다뤄도 되고, 어떤 파일 조합은 따로 봐야 하는지 정합니다
3. 그대로 쓰면 과밀해질 차원은 **상위 N개 + 기타** 규칙으로 줄입니다
4. 표에는 어떤 열만 남길지 정합니다
5. 집계가 필요한 경우 어떤 기준으로 묶을지 적습니다
6. 교차 파일 분석이 허용되는 조건과 금지 조건을 문서로 남깁니다

## 출력

`runs/latest/outputs/01b_dataset_prep.md`

## 출력 형식

```md
# 소스 정렬·집계 준비 메모

## 왜 이 단계가 필요한가
- ...

## 소스 역할 정리
| 파일 | 역할 | 함께 볼 수 있는 파일 | 주의 |
|------|------|----------------------|------|

## 관계 규칙
- 어떤 파일은 `join` 후보인지
- 어떤 파일은 `stack` 후보인지
- 어떤 파일은 `portfolio` 로 남겨야 하는지

## 차원 축약 규칙
- 어떤 차원은 상위 N개 + 기타로 줄일지
- 기준 N은 얼마인지

## 집계 규칙
- 파일별로 어떤 단위까지 묶어도 되는지
- 시간 단위가 있다면 어떤 단위로 맞출지

## 교차 파일 분석 허용 조건
- 공통 키:
- 공통 시간 축:
- 공통 단위:

## 표 구성 규칙
- 남길 열
- 숨길 열
- 표 행 cap / 열 cap

## 뒤 단계에 전달할 메모
- @eda:
- @metrics:
- @analysis:
- @dashboard:
```

## 원칙

- 원본 CSV는 바꾸지 않습니다
- 복잡한 입력을 사람이 읽을 수 있는 구조로 바꾸는 데 집중합니다
- 더 많이 보여주는 대신, 더 잘 고르는 쪽을 택합니다
- 결합이 애매하면 금지 규칙을 먼저 적습니다
