# 대시보드 디자인 스킬

반드시 아래 순서로 읽습니다.

1. `DESIGN.md`
2. `.claude/rules/input-contract.md`
3. `runs/latest/outputs/00_input_manifest.json`
5. 필요하면 `references/design-tokens.md`

## 핵심 원칙

- 어두운 관제 화면 톤
- 메인 차트 1개
- 보조 차트 2개 이하
- KPI 3~5개
- 탭 1~3개
- 표는 상한을 지키고 가로 스크롤 가능하게
- 여러 소스일 때는 출처를 짧게 드러내기

## 문구 원칙

- 상단 키커는 1단어로 끝냅니다
- 메인 제목은 짧고 강하게 씁니다
- 탭 라벨은 2~4자로 끊습니다
- 패널 제목은 2~8자 안팎으로 만듭니다
- 섹션 제목은 한 단어를 우선합니다
- 버튼 문구는 짧은 동작 라벨로 씁니다
- 설명은 제목 뒤에 길게 붙이지 말고 `page-subtitle`, `panel-problem`, `surface-desc`, `action-copy` 로 내립니다
- 소스 메타는 `3개 파일`, `비교 2종`, `결합 2종` 처럼 짧게 씁니다

## 구현 원칙

- CSS 변수 의미는 `DESIGN.md` 와 맞아야 합니다
- SVG는 가로폭에 맞게 꽉 차야 합니다
- 메타 정보는 모노 글꼴로 씁니다
- 시각적으로 가장 먼저 읽혀야 할 곳은 메인 차트입니다
- 문구가 길어지면 헤더 대신 본문을 정리합니다
- 라벨 길이 제약을 템플릿보다 우선합니다
- 패널마다 근거 소스가 보이도록 메타나 보조 문장을 배치합니다

## 완성 기준

- 탭을 눌렀을 때 버튼 자체만 봐도 역할이 구분되어야 합니다
- 제목만 훑어도 화면 구조가 읽혀야 합니다
- 긴 설명이 없어도 첫인상에서 정보 위계가 살아 있어야 합니다
- 화면이 미니멀해져도 설명 부족이 아니라 선택과 압축으로 느껴져야 합니다
- 여러 소스가 있어도 무엇을 섞었고 무엇을 분리했는지 바로 읽혀야 합니다
